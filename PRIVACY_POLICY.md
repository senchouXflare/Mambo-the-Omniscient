# Privacy Policy

**Last Updated:** December 9, 2024

## 1. Introduction

This Privacy Policy explains how our Discord Bot ("Bot", "Service", "we", "us", "our") collects, uses, stores, and protects your information when you use our Bot.

We are committed to protecting your privacy and ensuring transparency about our data practices.

## 2. Information We Collect

### 2.1 Automatically Collected Information

When you use the Bot, we automatically collect:

| Data Type | Description | Purpose |
|-----------|-------------|---------|
| **Discord User ID** | Your unique Discord identifier | Identify users, track permissions, prevent abuse |
| **Discord Username** | Your Discord display name | Display in leaderboards and commands |
| **Discord Server ID** | Server (guild) identifier | Server-specific configurations |
| **Channel ID** | Channel identifiers | Channel permission management |
| **Command Usage** | Commands used and timestamps | Functionality, analytics, debugging |

### 2.2 Information You Provide

When using Bot features, you may provide:

| Data Type | Description | Purpose |
|-----------|-------------|---------|
| **Club Names** | Names for clubs you create | Club identification and display |
| **Club URLs/IDs** | Links to uma.moe/chronogenesis profiles | Data synchronization |
| **Club Configuration** | Daily quotas, leaders, officers | Club management |
| **Member Names** | In-game player names | Leaderboard and statistics |
| **Fan Count Data** | Daily/monthly fan statistics | Tracking and analytics |

### 2.3 Information from Third Parties

We may receive information from:
- **uma.moe / chronogenesis.net** - Club and member data when syncing
- **Discord API** - User and server information as permitted

## 3. How We Use Your Information

We use collected information for:

### 3.1 Core Functionality
- Displaying leaderboards and statistics
- Managing club configurations
- Processing Bot commands
- User authentication and permissions

### 3.2 Service Improvement
- Debugging and error resolution
- Performance optimization
- Feature development

### 3.3 Security & Abuse Prevention
- Detecting and preventing misuse
- Enforcing Terms of Service
- Rate limiting and access control

### 3.4 Communication
- Responding to support requests
- Sending notifications (via Discord channels, if configured)

## 4. Data Storage and Security

### 4.1 Where We Store Data

Your data is stored in:

| Storage Type | Location | Purpose |
|-------------|----------|---------|
| **Google Sheets** | Google Cloud (US) | Club data, member stats, configurations |
| **Supabase Database** | Cloud (distributed) | Backup, historical data, fast queries |
| **Local Cache** | Bot server | Performance optimization (temporary) |
| **JSON Configuration Files** | Bot server | Settings, channel configs |

### 4.2 Security Measures

We implement appropriate security measures including:
- Encrypted connections (HTTPS/TLS)
- Service account authentication for Google Sheets
- Access controls and permission levels
- Environment variable protection for sensitive credentials
- No storage of Discord authentication tokens

### 4.3 Data Retention

| Data Type | Retention Period |
|-----------|-----------------|
| **Active club data** | As long as club is active |
| **Historical statistics** | Up to 12 months |
| **Command logs** | 30 days |
| **Channel configurations** | Until manually removed |
| **Cache data** | 24 hours (auto-expired) |

## 7. Data Deletion

### 7.1 Automatic Deletion
- Cache data: Expires after 24 hours
- Command logs: Purged after 30 days

### 7.2 User-Initiated Deletion
You can request data deletion by:
- Contacting us through the Support Server
- Server administrators can remove channel configurations
- Club managers can request club removal

### 7.3 Bot Removal
When the Bot is removed from a server:
- Server configurations may remain until manually deleted
- Club data persists unless deletion is requested
- You can request complete data deletion at any time
